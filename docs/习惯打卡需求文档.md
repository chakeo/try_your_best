# 习惯打卡应用 - 需求规格说明书（PRD）

## 1. 产品概述

### 1.1 产品定位
习惯打卡是一款帮助用户养成良好习惯的移动应用，通过每日打卡、连续天数统计、任务时间管理等功能，激励用户坚持完成目标习惯。

### 1.2 目标用户
- 希望养成良好习惯的个人用户
- 需要自我管理和时间规划的学生、职场人士
- 追求自我提升和目标达成的用户群体

### 1.3 核心价值
- 简单直观的打卡体验，降低坚持门槛
- 可视化的进度反馈，增强成就感
- 灵活的目标设置，适应不同习惯类型
- 本地数据存储，保护用户隐私

## 2. 功能需求

### 2.1 习惯管理模块

#### 2.1.1 创建习惯
**功能描述：** 用户可以创建新的习惯项目

**输入字段：**
- 习惯名称（必填，文本）
- 每日目标次数（必填，数字，默认1次）
- 目标天数（必填，数字，默认21天）

**业务规则：**
- 习惯名称不能为空
- 每日目标次数最小为1次
- 目标天数建议范围：7-365天
- 系统自动生成唯一ID和创建时间

**验收标准：**
- 成功创建后，习惯出现在列表首位
- 数据持久化到本地存储
- 界面自动刷新显示新习惯

#### 2.1.2 习惯列表展示
**功能描述：** 在主界面展示所有习惯的概览信息

**显示内容：**
- 习惯名称
- 当前连续天数
- 今日打卡进度（已打卡次数/目标次数）
- 打卡状态指示（已完成/未完成）

**交互行为：**
- 点击习惯卡片进入详情页
- 支持长按拖拽调整习惯顺序
- 下拉刷新数据

**业务规则：**
- 按用户自定义顺序排列
- 实时显示当日打卡状态
- 空状态提示"暂无习惯，点击右下角添加"

#### 2.1.3 每日打卡
**功能描述：** 用户可以对习惯进行打卡操作

**交互方式：**
- 点击习惯卡片上的打卡按钮
- 支持同一习惯每日多次打卡

**业务规则：**
- 每次打卡增加当日计数器
- 当日首次打卡时，将日期添加到已打卡日期列表
- 达到每日目标次数后，显示完成状态
- 打卡数据按日期（YYYY-MM-DD格式）存储

**验收标准：**
- 打卡后界面立即更新计数
- 连续天数实时重新计算
- 数据持久化保存

#### 2.1.4 习惯详情页
**功能描述：** 展示单个习惯的详细信息和历史数据

**显示内容：**
- 习惯基本信息（名称、每日目标、目标天数）
- 统计数据：
  - 当前连续天数
  - 历史最长连续天数
  - 累计打卡天数
- 月度日历视图：
  - 显示当月所有日期
  - 已打卡日期高亮显示
  - 支持左右切换月份
  - 显示农历日期

**交互功能：**
- 删除习惯（需二次确认）
- 月份切换查看历史记录

**业务规则：**
- 连续天数从今天往前计算，中断则重新开始
- 最长连续天数统计历史最佳记录
- 删除操作不可恢复

#### 2.1.5 习惯排序
**功能描述：** 用户可以自定义习惯在列表中的显示顺序

**交互方式：**
- 长按习惯卡片进入拖拽模式
- 拖动到目标位置后释放

**业务规则：**
- 排序结果实时保存
- 重启应用后保持用户设置的顺序

### 2.2 任务管理模块

#### 2.2.1 创建任务
**功能描述：** 用户可以创建需要投入时间的任务

**输入字段：**
- 任务名称（必填）
- 目标时长（小时，必填）
- 截止日期（必填）

**业务规则：**
- 任务名称不能为空
- 目标时长自动转换为分钟存储
- 截止日期不能早于当前日期
- 新任务默认状态为"进行中"

#### 2.2.2 任务列表
**功能描述：** 展示所有任务及其进度

**显示内容：**
- 任务名称
- 进度条（已完成时长/目标时长）
- 剩余天数
- 任务状态（进行中/已完成/已放弃）

**顶部图表：**
- 时间分布饼图
- 显示各任务时间占比
- 可视化时间分配情况

**交互行为：**
- 点击任务卡片进入详情页
- 点击"开始"按钮快速进入计时

#### 2.2.3 任务详情与计时
**功能描述：** 记录任务执行时间

**计时功能：**
- 开始计时按钮
- 实时显示计时器
- 结束计时并保存时间段

**显示内容：**
- 任务基本信息
- 总进度和剩余时间
- 历史时间段列表（开始时间、结束时间、时长）

**业务规则：**
- 每个时间段独立记录
- 自动累加到总时长
- 进度百分比 = 总时长 / 目标时长

#### 2.2.4 任务状态管理
**功能描述：** 管理任务的生命周期

**状态类型：**
- 进行中（active）：默认状态
- 已完成（completed）：达成目标
- 已放弃（abandoned）：中止任务

**状态切换：**
- 用户手动标记完成或放弃
- 状态变更后保留历史数据

### 2.3 界面导航

#### 2.3.1 底部导航栏
**功能描述：** 在习惯和任务两个模块间切换

**导航项：**
- 习惯（图标：check_circle）
- 任务（图标：timer）

**交互行为：**
- 点击切换页面
- 保持各页面状态

## 3. 数据模型

### 3.1 习惯（Habit）
```
{
  id: String,              // 唯一标识
  name: String,            // 习惯名称
  checkedDates: List<String>,  // 已打卡日期列表 ['2026-03-25']
  dailyGoal: int,          // 每日目标次数，默认1
  targetDays: int,         // 目标天数，默认21
  checkCounts: Map<String, int>  // 每日打卡次数 {'2026-03-25': 3}
}
```

### 3.2 任务（Task）
```
{
  id: String,              // 唯一标识
  name: String,            // 任务名称
  targetMinutes: int,      // 目标时长（分钟）
  deadline: DateTime,      // 截止日期
  status: TaskStatus,      // 状态枚举
  sessions: List<TimeSession>  // 时间段列表
}
```

### 3.3 时间段（TimeSession）
```
{
  id: String,              // 唯一标识
  startTime: DateTime,     // 开始时间
  endTime: DateTime,       // 结束时间
  durationMinutes: int     // 时长（分钟）
}
```

## 4. 技术实现

### 4.1 技术栈
- **框架：** Flutter 3.10.7+
- **语言：** Dart
- **存储：** SharedPreferences（本地持久化）
- **依赖库：**
  - intl: 日期格式化
  - lunar: 农历日期显示

### 4.2 数据存储
- 使用 SharedPreferences 存储 JSON 格式数据
- 习惯数据键：`habits`
- 任务数据键：`tasks`
- 数据在每次修改后立即保存

### 4.3 核心算法

#### 连续天数计算
- 从今天开始往前遍历已打卡日期
- 每遇到连续的日期，计数器+1
- 遇到中断日期则停止计算
- 返回连续天数

#### 最长连续天数计算
- 遍历所有已打卡日期
- 计算每个连续区间的长度
- 返回最大值
